# 什么是超链接？
超链接是HTML文档的功能，当点击或以其他方式激活时，浏览器会引导浏览器导航到其他文档或资源，有时是文档的特定部分。 超链接是网络上最令人兴奋的创新之一。 它们自互联网诞生以来就是其特征，也是使网络成为网络的关键。网络上的每个资源都有一个地址，称为URL（统一资源定位码），超链接指向该地址。
# 链接的结构
通过将文本或其他内容包裹在`<a>`元素中，并使用包含网页地址的href属性（也称为超文本引用或目标）来创建基本链接

## 区块级链路
几乎任何内容都可以被制作成链接，甚至是区块级元素。 如果你想把标题元素做成链接，那么用锚点元素包裹它
```
<a href="https://developer.mozilla.org/en-US/">
  <h1>MDN Web Docs</h1>
</a>
<p>
  Documenting web technologies, including CSS, HTML, and JavaScript, since 2005.
</p>
```

## 图片链接:
要将图片转换为链接，将`<img>`元素包裹为`<a>`元素
```
<a href="https://developer.mozilla.org/en-US/">
  <img src="mdn_logo.svg" alt="MDN Web Docs" />
</a>
```

## 添加带有标题属性的支持信息:
可以为链接添加属性; 标题包含关于链接的额外信息，比如页面包含哪些类型的信息，或网站上需要注意的事项
```
<p>
  I'm creating a link to
  <a
    href="https://www.mozilla.org/en-US/"
    title="The best place to find more information about Mozilla's mission and how to contribute">
    the Mozilla homepage</a
  >.
</p>
```

tip：
```
正确代码:
  <li>
    <a
      href="https://en.wikipedia.org/wiki/Eastern_gray_squirrel"
      title="Sciurus carolinensis">
      Eastern gray squirrel
    </a>
  </li>
错误代码:
 <a
  href=""
    title="">
      <li>
      Eastern gray squirrel
      </li>
 </a>

```
`<a>`不能包裹`<li>`,以下是理由：
1. `<ul>` (无序列表) 里面只能包含 `<li>` (列表项) 或者 `<script>` 标签。
2. `<a>` 标签在这里被视为“非法”的直接子元素。
3. 虽然浏览器（如 Chrome）很聪明，它会尝试“修复”你的代码（通常会把 `<a>` 踢出 `<ul>` 或者把 `<li>` 强行拉出来），但这会导致 DOM 结构混乱，甚至导致样式失效或点击区域异常。

# 关于URL和路径的快速入门介绍
链接目标是网址。URL，或称统一资源定位器（Uniform Resource Locator），是一串定义某物在网络上位置的文本。例如，Mozilla的英文主页位于 https://www.mozilla.org/en-US/

网络服务器接收 URL 请求并响应相应资源。大多数资源以文件形式存储在服务器的文件系统中，因此这些资源的URL通常类似于文件路径
## 同一个目录:
如果想在顶层内包含指向一个文件的超链接，可以只指定路径为想链接的文件名，因为它和当前文件在同一个目录里
```
<p>
  Want to contact a specific staff member? Find details on our
  <a href="contacts.html">contacts page</a>.
</p>
```

也可以用一个点和斜杠开始一条路径，指向同一目录内的文件
```
<p>
  Want to contact a specific staff member? Find details on our
  <a href="./contacts.html">contacts page</a>.
</p>
```
## 下层进入子目录
如果想在顶层index.html中包含指向projects/index.html的超链接，需要先进入projects目录，然后再指定想链接的文件。这通过指定目录名称，然后斜杠，再指定文件名称来实现。
```
<p>
  Want to contact a specific staff member? Find details on our
  <a href="contacts.html">contacts page</a>.
</p>
```
## 回到父级目录:
如果你想在projects/index.html里加指向pdfs/project-brief.pdf的超链接，你得先往上爬一个目录层，然后再回到pdfs目录。要向上查找目录，你用两个点——..
```
<p>A link to my <a href="../pdfs/project-brief.pdf">project brief</a>.</p>
```

## 相对于根目录的链接
如果想创建一个指向特定位置且不会因移动链接文件而断裂的链接，可以通过在路径开头加一个斜杠来实现——这表示路径从网站的根目录开始
```
<p>A link to my <a href="/pdfs/project-brief.pdf">project brief</a>.</p>
```
现在路径总是从根目录（creating-hyperlinks）出发，前往pdfs目录，找到project-brief.pdf文件。如果把链接文件移到其他位置，比如a/b/c/d/e/index.html，这个方法依然有效。

如果把链接project-brief.pdf文件移到别处，链接依然会被破坏

1. 绝对路径：指向由其在网站（或网络其他地方）绝对位置定义的位置。例如，可以创建一个绝对链接，始终指向相对于站点根目录的同一位置，就像我们之前看到的：/pdfs/project-brief.pdf
2. 相对路径：指向相对于所链接文件的位置。在之前的例子中，用projects/index.html创建了当前文件与位于projects子目录中的名为index.html的文件之间的相对链接。如果把当前文件移动到另一个位置，路径仍然是相对于该文件，但指向的绝对位置会不同。

这些术语的使用并不总是一致。例如，/pdfs/project-brief.pdf 相对于当前文件的位置是绝对的，但相对于域名是相对的

## 带完整网址的链接
可以指定一个完整的 URL 作为路径，无论在哪里使用，它都会指向网页上的同一位置
```
<a href="https://www.example.com/projects/">projects</a>
```
## 内部与外部链接
- 当链接指向你网站上的某个资源时，称为内部链接
在指定内部链接时，可以使用相对路径或绝对路径，或者完整网址
推荐不要使用完整域名，因为便于携带。如果指定https://www.example.com/projects/，它总会链接到https://www.example.com/projects/。如果将网站迁移到其他域名，例如another-example.com，所有完整的URL链接都需要更改。

如果指定路径如/projects，它们依然能工作，因为它们仍然相对于目录结构存在。

- 当链接指向不同网站的资源时，称为外部链接
在指定外部链接时，必须始终包含完整的 URL 作为路径

## 文献片段
可以链接到HTML文档的特定部分，称为文档片段，而不仅仅是文档顶部。 文档中带有id属性的元素会自动创建一个可链接的文档片段

最常见的用例是链接到特定标题:
```
<h2 id="mailing_address">Mailing address</h2>
```

要链接到那个具体id，在文件路径末尾加上哈希/井符号（#）再加id

```
<p>
  Want to write us a letter? Use our
  <a href="contacts.html#mailing_address">mailing address</a>.
</p>
```

甚至可以单独使用文档片段引用链接到当前文档的其他部分
```
<p>
  The <a href="#mailing_address">company mailing address</a> can be found at the
  bottom of this page.
</p>
```

# URL如何转换为文件路径？
没有任何网页内容能直接访问服务器的文件系统。

目前看到的服务器示例是创建一个静态网站。 服务器只需获取URL中的路径名部分，直接在其文件系统中查找对应的文件。

>注： 许多服务器是实时生成URL内容，而不是从静态文件中检索。如果用的是Web框架，源代码目录可能和部署到服务器的版本有很大不同。在管理自己的网站时，需要了解构建工具和服务器配置，才能知道URL是如何映射到源文件的

如果用示例网站目录作为根节点，并设置网站域名为example.com，pdfs/project-brief.pdf文件就会在https://www.example.com/pdfs/project-brief.pdf上提供

所有链接均相对于当前文档的URL解析，因此：
- 对于https://example.com域上的所有页面，链接到/pdfs/project-brief.pdf总是会生成指向路径名为/pdfs/project-brief.pdf的https://www.example.com/pdfs/project-brief.pdf的链接。服务器会在根目录中寻找pdfs目录，然后在该目录中寻找project-brief.pdf文件。
- 指向projects/index.html的链接会创建指向https://www.example.com/projects/index.html的链接，但前提是必须包含在根目录内的文件中，例如顶层的index.html文件，或contacts.html。(本质当做相对路径来处理)例如，如果把它包含在pdfs/index.html的HTML文件里，它会链接到路径名/pdfs/projects/index.html的https://www.example.com/pdfs/projects/index.html，而这个路径不存在，所以你最终会看到一个断链。

## 默认index.html页面
引用index.html文件时，通常不需要在 URL/路径中包含index.html，因为当文件名未指定时，服务器会寻找名为 index.html 的默认落地页

再看projects/index.html路径示例，只需将路径写成projects，这样就能创建一个指向https://www.example.com/projects/index.html的链接。在导航到页面时，我们可以把URL写成https://www.example.com/projects/，这样也能到达正确的位置
>URL末尾的斜杠（/）非常重要。有了它，projects/index.html中contacts.html的相对链接会解析为https://www.example.com/projects/contacts.html（该链接在同一目录中）。没有它，URL将被视为文件，相关链接解析为https://www.example.com/contacts.html

# 链接最佳实践:
## 使用清晰的链接措辞：
在页面上放链接很容易。这还不够。我们需要让所有读者都能访问我们的链接，无论他们当前的语境如何，以及他们偏好哪种工具
tips：
1. 不要在链接文本中重复URL——URL看起来很难看，当屏幕阅读器逐字母朗读时听起来更难看
2. 不要在链接文本里说“链接”或“链接到”——那只是噪声。屏幕阅读器告诉人们有链接。 视觉用户也会知道有链接，因为链接通常用不同颜色样式并下划线
3. 保持链接文本尽可能简短——这很有帮助，因为屏幕阅读器需要解读整个链接文本。
4. 尽量减少同一文本的多份副本被链接到不同地点的情况。 如果屏幕上出现了标签为“点击这里”、“点击这里”、“点击这里”的断章取义链接列表，可能会给屏幕阅读器用户带来问题。

# 链接到非HTML资源——留下清晰的指示
当链接到当前页面不会作为“正常导航”打开的资源时，应该在链接文本中添加明确的措辞，说明接下来会发生什么。例如，如果你正在下载或流媒体某个资源，或者链接会弹出弹窗或产生其他可能意想不到的效果，这些都应该在正文中说明。这对于低带宽连接用户尤为重要，他们可能想避免下载数兆字节的资源。这也有助于为屏幕阅读器用户设定预期，因为他们可能不会知道其他情况。
```
<p>
  <a href="/large-report.pdf" download>
    Download the sales report (PDF, 10MB)
  </a>
</p>

<p>
  <a href="https://www.example.com/video-stream/" target="_blank">
    Watch the video (stream opens in separate tab, HD quality)
  </a>
</p>
```

## 链接下载时使用下载属性
当你链接到需要下载而非浏览器打开的资源时，可以使用download属性提供默认存档文件名
```
<a
  href="https://download.mozilla.org/?product=firefox-latest-ssl&os=win64&lang=en-US"
  download="firefox-latest-64bit-installer.exe">
  Download Latest Firefox for Windows (64-bit) (English, US)
</a>

```

## 何时在新标签页中打开链接:
链接默认会在与页面相同的标签页中打开，用户可以通过浏览器的返回按钮返回上一页。然而，许多网站（包括MDN）会选择在新标签页中打开某些链接，尤其是外部链接。这通过将target属性设置为"_blank"来实现。
```
Firefox is developed by the
<a href="https://www.mozilla.org/en-US/" target="_blank">Mozilla Foundation</a>.
```
一种常见的做法是在新标签页中打开外部链接，在同一标签页中打开内部链接。 有些设计师更喜欢在同一个标签页中打开所有链接。 如果你在新标签页中打开链接，建议你为这些链接提供提示，比如在链接文本旁边放置图标

# 电子邮件链接
可以创建链接或按钮，点击后打开新的外发邮件，而不是链接到资源或页面。 这是通过`<a>`元素和mailto: URL方案实现的

最基本且常用的形式是mailto:链接表示收件人的电子邮件地址
```
<a href="mailto:nowhere@mozilla.org">Send email to nowhere</a>
```
事实上，电子邮件地址是可选的。如果省略了它，而href是“mailto：”，用户的邮件客户端会打开一个没有目的地的新寄出邮件窗口。 这通常作为“分享”链接，用户可以点击发送邮件到自己选择的地址。

## 具体说明：
除了电子邮件地址，还可以提供其他信息。事实上，任何标准邮件头字段都可以添加到提供的mailto URL中。 其中最常用的是“subject主题”、“cc抄送人”和“body正文”（正文不是真正的头字段，但允许为新邮件指定简短的内容信息）。 每个字段及其值都指定为查询词
```
<a
  href="mailto:nowhere@mozilla.org?cc=name2@rapidtables.com&bcc=name3@rapidtables.com&subject=The%20subject%20of%20the%20email&body=The%20body%20of%20the%20email">
  Send mail with cc, bcc密送人, subject and body
</a>
```

>注： 每个字段的值必须用非打印字符（如制表符、回车和换页符等隐形字符）和空格进行百分比脱离的URL编码。 另外，请注意使用问号（?）分隔主URL与字段值，以及用&符号（&）分隔mailto:URL中的每个字段。 这是标准的 URL 查询符号

>%20是空格，内容里有空格或特殊符号，都要把它们转换成 %20 这种编码格式
