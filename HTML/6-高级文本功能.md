## 引言:
HTML包含可用于标记引文的功能;用哪个元素取决于在标记块还是内联引文

### 引用块:
如果某个段落层级内容（无论是一段、多段、列表等）是从其他地方引用的，你应该用`<blockquote>`元素包裹以表示，并在引用属性中包含指向引用来源的URL
```
<p>
  The <strong>HTML <code>&lt;blockquote&gt;</code> Element</strong> (or
  <em>HTML Block Quotation Element</em>) indicates that the enclosed text is an
  extended quotation.
</p>
                                        |
                                        |
                                        |
                                        |
                                        |
                                        |
                                       \/
<p>Here is a blockquote:</p>
<blockquote
  cite="https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/blockquote">
  <p>
    The <strong>HTML <code>&lt;blockquote&gt;</code> Element</strong> (or
    <em>HTML Block Quotation Element</em>) indicates that the enclosed text is
    an extended quotation.
  </p>
</blockquote>
```
>`<code>`标签在 HTML 中专门用于标记计算机代码或计算机相关的文本

>浏览器默认会将`<code>`包裹的内容渲染为等宽字体,并且通常字号会稍微小一点

浏览器默认样式会将此段(bockquote内部的)显示为**缩进段落**，作为引用的指示

### 内文引言：
内联引号的工作原理完全相同，只是它们使用了`<q>`元素
```
<p>
  The quote element — <code>&lt;q&gt;</code> — is
  <q
    cite="https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/q">
    intended for short quotations that don't require paragraph breaks.
  </q>
</p>
```
浏览器默认样式会将此表示为**普通文本**，并用**引号**表示引用

### 引用:
cite**属性**的内容听起来很有用，但遗憾的是浏览器、屏幕阅读器等并未真正利用它。没有办法让浏览器显示cite的内容，除非用JavaScript或CSS自己写解决方案。如果想在页面上提供引用来源，需要通过链接或其他合适的方式在正文中提供

虽然有`<cite>`元素，但这里包含被引用资源的标题，例如书籍名称。不过，没有理由你不能以某种方式将`<cite>`中的文本链接到引用来源

>引用文`<cite>`默认采用斜体字体样式

## 缩写:
在网上浏览时，你会遇到另一个相当常见的元素是`<abbr>`——它用来包裹缩写或缩略词。在包含其中任一时，首次使用时请以纯文字完整扩展该术语，并附上标记缩写的`<abbr>`。这为用户代理提供了提示，了解如何在向所有用户说明缩写含义的同时，宣布和展示内容

如果除了缩写外提供扩展意义不大，且缩写或缩写是一个相当简短的术语，请提供该项的完整扩展，作为**title属性**的值

```
<p>
  We use <abbr>HTML</abbr>, Hypertext Markup Language, to structure our web
  documents.
</p>

<p>
  I think <abbr title="Reverend">Rev.</abbr> Green did it in the kitchen with
  the chainsaw.
</p>

```

## 标记联系方式:
HTML有一个标记联系方式的元素——`<address>`

><address>Chris Mills, Manchester, The Grim North, UK</address>


还可以包含更复杂的标记和其他联系方式
```
<address>
  <p>
    Chris Mills<br />
    Manchester<br />
    The Grim North<br />
    UK
  </p>

  <ul>
    <li>Tel: 01234 567 890</li>
    <li>Email: me@grim-north.co.uk</li>
  </ul>
</address>
```
如果链接页面包含联系方式，这样的信息也可以接受:
```
<address>
  Page written by <a href="../authors/chris-mills/">Chris Mills</a>.
</address>
```

## 上标和下标；
在标记日期、化学式和数学方程等项时，有时需要使用上标和下标，以确保它们的正确含义。`<sup>`(上标)和`<sub>`(下标)元素负责这项工作
```
<p>My birthday is on the 25<sup>th</sup> of May 2001.</p>
<p>
  Caffeine's chemical formula is
  C<sub>8</sub>H<sub>10</sub>N<sub>4</sub>O<sub>2</sub>.
</p>
<p>If x<sup>2</sup> is 9, x must equal 3 or -3.</p>
```

## 表示计算机代码：
- `<code>`：用于标记通用的计算机代码。

- `<pre>`：关于保留空白（通常是代码块）——如果你在文本中使用缩进或多余空白，浏览器会忽略它，你在渲染页面上看不到。但如果你用`<pre></pre>`标签包裹文本，空白区域会和你在文本编辑器中看到的完全一样。

- `<var>`：专门标记**变量名**。

- `<kbd>`：用于标记输入计算机的**键盘（及其他类型）输入**。

- `<samp>`：用于标记计算机程序的**输出**。

```
<pre><code>const para = document.querySelector('p');

para.onclick = function() {
  alert('Owww, stop poking me!');
}</code></pre>

<p>
  You shouldn't use presentational elements like <code>&lt;font&gt;</code> and
  <code>&lt;center&gt;</code>.
</p>

<p>
  In the above JavaScript example, <var>para</var> represents a paragraph
  element.
</p>

<p>Select all the text with <kbd>Ctrl</kbd>/<kbd>Cmd</kbd> + <kbd>A</kbd>.</p>

<pre>$ <kbd>ping mozilla.org</kbd>
<samp>PING mozilla.org (63.245.215.20): 56 data bytes
64 bytes from 63.245.215.20: icmp_seq=0 ttl=40 time=158.233 ms</samp></pre>
```

## 标记时间和日期
HTML还提供了`<time>`元素，用于以机器可读格式标记时间和日期
```
<time datetime="2016-01-20">20 January 2016</time>
```
不同的时间写法很难被电脑识别,`<time>`元素允许为此附上一个明确、机器可读的时间/日期。

还有许多其他选项：
```
<!-- Standard simple date -->
<time datetime="2016-01-20">20 January 2016</time>
<!-- Just year and month -->
<time datetime="2016-01">January 2016</time>
<!-- Just month and day -->
<time datetime="01-20">20 January</time>
<!-- Just time, hours and minutes -->
<time datetime="19:30">19:30</time>
<!-- You can do seconds and milliseconds too! -->
<time datetime="19:30:01.856">19:30:01.856</time>
<!-- Date and time -->
<time datetime="2016-01-20T19:30">7.30pm, 20 January 2016</time>
<!-- Date and time with timezone offset -->
<time datetime="2016-01-20T19:30+01:00">
  7.30pm, 20 January 2016 is 8.30pm in France
</time>
<!-- Calling out a specific week number -->
<time datetime="2016-W04">The fourth week of 2016</time>
```
