## 什么是HTML头？:
HTML 标题是` <head>` 元素的内容。与`<body>`元素的内容（在浏览器加载时会显示在页面上）不同，头部的内容不会显示在页面上。相反，负责人的工作是包含关于文档的元数据。

### 添加标题
<标题>元素的实际应用——它可以用来为文档添加标题。不过这容易与h1元素混淆，h1用于为正文内容添加顶层标题——有时也称为页面标题。

## 元数据：`<meta>`元素
元数据是描述数据的数据，HTML有一种“官方”的方式向文档添加元数据——`<meta>`元素

### 指定文档的字符编码：
```
<meta charset="utf-8" />

```
该元素指定文档的字符编码——即文档允许使用的字符集。 是一个通用字符集，几乎包含任何人类语言中的字符

>注： 有些浏览器（比如 Chrome）会自动修复错误编码，所以根据你使用的浏览器，你可能不会遇到这个问题。不过你还是应该在页面上设置编码，以避免在其他浏览器出现潜在问题。

### 添加作者和描述:
`<meta>` name content
- name 指定其元元素类型;它包含的信息类型
- content 指定实际的元内容

两个有用的元元素可以定义页面作者，并提供页面的简明描述。
```
<meta name="author" content="Chris Mills" />
<meta
  name="description"
  content="The MDN Web Docs Learning Area aims to provide
complete beginners to the Web with all they need to know to get
started with developing websites and applications." />

```
>description也可以用于搜索引擎结果页面，充当关键词(`keyword`已经被废弃)

### 其他类型的元数据
在网络上游走时，会发现其他类型的元数据。会在网站上看到的许多功能都是专有的，旨在为某些网站（如社交网络网站）提供可用的特定信息。

例如，开放图数据是Facebook发明的一种元数据协议，旨在为网站提供更丰富的元数据
```
<meta
  property="og:image"
  content="https://developer.mozilla.org/mdn-social-share.png" />
<meta
  property="og:description"
  content="The Mozilla Developer Network (MDN) provides
information about Open Web technologies including HTML, CSS, and APIs for both websites
and HTML Apps." />
<meta property="og:title" content="Mozilla Developer Network" />

```
其中一个效果是，当你在Facebook上链接到MDN Web Docs时，链接会连同图片和描述一起出现：为用户带来更丰富的体验

## 将CSS和JavaScript应用于HTML应用
现代几乎所有网站都会用CSS来让它们看起来酷炫，JavaScript则支持互动功能，比如视频播放器、地图、游戏等等。这些通常分别应用于网页，分别使用`<link>`元素和`<script>`元素。

`<link>`元素应始终放在文档的头部。这需要两个属性，表示它是文档样式表和包含样式表文件的路径
```
<link rel="stylesheet" href="my-css-file.css" />

```
`<script>`元素也应放在脑袋中，并包含包含你想加载的JavaScript路径的属性，以及一个布尔属性，指示浏览器在页面解析完成HTML后加载JavaScript。这个属性很有用，因为它保证 HTML 在 JavaScript 运行前就已经加载好，这样你就不会因为 JavaScript 试图访问页面上还不存在的 HTML 元素而出错。
```
<script src="my-js-file.js" defer></script>
```
>注： 该元素看起来像空元素，但其实不是，因此需要一个关闭标签。你也可以选择把脚本放进元素里，而不是指向外部脚本文件

## 设定文档的主要语言
可以（而且真的应该）设定页面的语言。这可以通过在开启的HTML标签中添加lang属性来实现
```
<html lang="en-US">
  …
</html>
```
这在很多方面都很有用。如果HTML文档语言设置好（例如能正确显示在特定语言的结果中），搜索引擎会更有效地索引它，并且对使用屏幕阅读器的视障人士也很有用（例如，“six”这个词在法语和英语中都有，但发音不同）

还可以设置文档的子部分识别为不同语言。例如可以将日语部分设置为识别为日语
```
<p>Japanese example: <span lang="ja">ご飯が熱い。</span>.</p>
```
