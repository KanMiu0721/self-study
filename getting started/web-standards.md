## 工具箱
- TCP/IP：传输控制协议和互联网协议（TCP/IP）是定义数据如何在互联网中传输的通信协议。这就像那些运输机制，让你下单、去商店、购买商品。在我们的例子中，这就像汽车或自行车（或者你在路上可能以其他方式行驶）。
- DNS：域名系统（DNS）就像网站的通讯录。当你在浏览器中输入网页地址时，浏览器会查看DNS以查找网站的IP地址——服务器实际所在的地址——然后才能检索该网站（详见下文DNS解释）。浏览器需要知道网站所在的服务器，以便将HTTP消息发送到正确的位置（见下文）。这就像你去店铺之前先查地址一样。
- HTTP，文件，你的互联网链接等等

### DNS 
DNS：真实的网址（URL）并不是你在地址栏输入的那些漂亮、令人难忘的字符串，用来查找你喜欢的网站。它们是特殊数字，看起来如下：.192.0.2.172

这被称为IP地址，代表网络上的唯一位置
>PS:大型网站通常会在多个服务器上发布，以便全球不同用户能够高效地加载。因此，IP地址可能会因你所在的位置而异。

### HTTP
HTTP 使用一种简单的动词语言来执行诸如发送请求等动作。HTTP GET 方法通常用于发出上述类型的 HTTP 请求。例如，MDN主页的请求可能如下：
```
 GET /en-US/ HTTP/2
 Host: developer.mozilla.org
```
服务器发送的回复可能如下（简略版）：
```
HTTP/2 200

date: Tue, 11 Feb 2025 11:13:30 GMT
expires: Tue, 11 Feb 2025 11:40:01 GMT
server: Google frontend
last-modified: Tue, 11 Feb 2025 00:49:32 GMT
ETag: "65f26b7f6463e2347f4e5a7a2adcee54"
content-length: 45227
content-type: text/html

<!doctype html> ... (the 45227 bytes of the requested web page HTML)
```
- HTTP/2 200

>服务器用来发送响应的HTTP版本（此处为HTTP/2），后接一个状态码，表示请求是否成功。 表示成功为200

- date, expires, etc.date、expires等等。

>HTTP头包含关于响应的额外信息（注意请求也可以有头部），这些头部提供额外信息和/或修改响应的行为。

- <!doctype html>, etc.<!doctype html>，等等。

>响应正体，在此中包含MDN主页的HTML文档。

#### 其他状态码：
- 301:请求的资源已被永久迁移到新地点，该位置已在回复中提供。这用于在内容被移动时重定向。

- 403:服务器不会让客户端访问请求的资源。这通常发生在服务器知道客户端是谁，但没有权限访问请求页面时。

- 404:服务器无法找到请求的资源。如果URL错误或内容被删除且未设置重定向，通常会返回该状态。

- 503:由于服务器存在问题，请求无法处理。当服务器因维护离线时，这种情况很常见，而且通常只是临时的。
