## 工具箱
- TCP/IP：传输控制协议和互联网协议（TCP/IP）是定义数据如何在互联网中传输的通信协议。这就像那些运输机制，让你下单、去商店、购买商品。在我们的例子中，这就像汽车或自行车（或者你在路上可能以其他方式行驶）。
- DNS：域名系统（DNS）就像网站的通讯录。当你在浏览器中输入网页地址时，浏览器会查看DNS以查找网站的IP地址——服务器实际所在的地址——然后才能检索该网站（详见下文DNS解释）。浏览器需要知道网站所在的服务器，以便将HTTP消息发送到正确的位置（见下文）。这就像你去店铺之前先查地址一样。
- HTTP，文件，你的互联网链接等等

### DNS 
DNS：真实的网址（URL）并不是你在地址栏输入的那些漂亮、令人难忘的字符串，用来查找你喜欢的网站。它们是特殊数字，看起来如下：.192.0.2.172

这被称为IP地址，代表网络上的唯一位置
>PS:大型网站通常会在多个服务器上发布，以便全球不同用户能够高效地加载。因此，IP地址可能会因你所在的位置而异。

### HTTP
HTTP 使用一种简单的动词语言来执行诸如发送请求等动作。HTTP GET 方法通常用于发出上述类型的 HTTP 请求。例如，MDN主页的请求可能如下：
```
 GET /en-US/ HTTP/2
 Host: developer.mozilla.org
```
服务器发送的回复可能如下（简略版）：
```
HTTP/2 200

date: Tue, 11 Feb 2025 11:13:30 GMT
expires: Tue, 11 Feb 2025 11:40:01 GMT
server: Google frontend
last-modified: Tue, 11 Feb 2025 00:49:32 GMT
ETag: "65f26b7f6463e2347f4e5a7a2adcee54"
content-length: 45227
content-type: text/html

<!doctype html> ... (the 45227 bytes of the requested web page HTML)
```
- HTTP/2 200

>服务器用来发送响应的HTTP版本（此处为HTTP/2），后接一个状态码，表示请求是否成功。 表示成功为200

- date, expires, etc.date、expires等等。

>HTTP头包含关于响应的额外信息（注意请求也可以有头部），这些头部提供额外信息和/或修改响应的行为。

- <!doctype html>, etc.<!doctype html>，等等。

>响应正体，在此中包含MDN主页的HTML文档。

#### 其他状态码：
- 301:请求的资源已被永久迁移到新地点，该位置已在回复中提供。这用于在内容被移动时重定向。

- 403:服务器不会让客户端访问请求的资源。这通常发生在服务器知道客户端是谁，但没有权限访问请求页面时。

- 404:服务器无法找到请求的资源。如果URL错误或内容被删除且未设置重定向，通常会返回该状态。

- 503:由于服务器存在问题，请求无法处理。当服务器因维护离线时，这种情况很常见，而且通常只是临时的。

### URL
URL 是一个网页地址加上一个协议：例如，如果你在浏览器中打开一个新标签页，在地址栏输入并点击 /，你会被重定向到类似以下的 URL：
```
https://developer.mozilla.org/en-US/
```
- https:用于发送请求的协议
  
- developer.mozilla.org:URL的域名，代表你连接服务器的顶层位置

- /en-US/:你正在访问服务器上资源的路径

## 网络标准：
网络标准是我们用来构建网站的技术

## 静态与动态：
客户端和服务器端语言的另一种描述方式是静态和动态：

- 服务器上存储一个普通的HTML文件。当请求时，它会不变地交付给客户端，并由浏览器渲染。由于它不变，被称为“静态”。
- 当服务器端代码（例如Python脚本或 ASP.NET 页面）生成包含数据的HTML并将该HTML返回给客户端时，HTML内容会根据服务器端代码的功能而变化。因此，它被称为“动态”。

## 网页渲染：
 当用户通过点击链接或在浏览器地址栏输入网页地址访问新网页时，会发送多个HTTP请求，并以HTTP响应形式返回多个文件。这些回复中收到的文件会被浏览器处理，并整合成用户可以互动的网页。将这些部分组装成网页的过程称为**渲染**。

###  处理HTML:
首先，包含网页内容并定义其结构的HTML文件会被浏览器接收并解析。浏览器将其转换为一种称为**DOM树**（文档对象模型）的树状结构。DOM 表示计算机内存中的 HTML 文档结构。
- eg:
```
<p>
  Let's use:
  <span>HTML</span>
  <span>CSS</span>
  <span>JavaScript</span>
</p>
```
HTML中的每个元素、属性和文本段都成为树结构中的一个DOM节点。节点由它们与其他DOM节点的关系定义。有些元素是子节点的父节点，子节点又有兄弟节点。浏览器会解析这些 HTML，并从中生成以下 DOM 树：
```
P
├─ "Let's use:"
├─ SPAN
|  └─ "HTML"
├─ SPAN
|  └─ "CSS"
└─ SPAN
    └─ "JavaScript"
```
在这个DOM树中，对应元素的节点是父节点。其子节点包括一个文本节点和对应我们元素的三个节点。这些节点也是父节点，文本节点是其子节点。

### 解析CSS和页面渲染:
- 浏览器解析页面上的CSS（无论是包含在HTML文件中，还是从外部样式表获取的），并根据应用到哪些HTML元素（在DOM中称为节点的项）将不同的CSS样式规则分类到不同的“桶”中。浏览器随后根据需要将样式附加到不同元素上（这一中间步骤称为渲染树）。
- 渲染树在规则应用后应出现的结构中布局。这包括将嵌入页面中的任何图片和其他媒体文件。
- 页面的视觉显示会在屏幕上显示（这一阶段称为绘画）。

### JavaScript 处理:
CSS 处理完成后，页面上发现的任何 JavaScript（无论是包含在 HTML 文件中，还是从外部脚本文件获取）都会被解析、解释、编译并执行。这通常发生在最终页面渲染完成之前的某个阶段——毕竟，某些JavaScript可能会影响渲染，例如通过向DOM添加节点或修改现有节点。
